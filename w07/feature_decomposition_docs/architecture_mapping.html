Justification

Given the moderate number of features and the solo developer context, a feature-first folder structure is recommended. This approach allows for clear separation of concerns, easier navigation, and more maintainable code as each feature is encapsulated with its UI, logic, and data access. Angular and Node.js both support modular development, and PostgreSQL fits well with repository-based data access. Features are likely to change independently, and a feature-first structure minimizes cross-feature coupling and simplifies refactoring. For a university project, this structure is easier to scale and reason about, and aligns with modern best practices for full-stack development. Layer-first is more suitable for large teams or highly reusable shared logic, which is not the case here.
